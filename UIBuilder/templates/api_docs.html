Data API

Read-only JSON backed by this app’s SQLite catalog and ingested feeds. Examples use placeholders — swap in your own values. Downloadable reports and AI text omit internal route strings; use this page for query shapes.

API access

Generate a personal key to call /api/news/feed with saved-home ranking and an optional house_name that matches a profile nickname. Prefer the X-API-Key header (or Authorization: Bearer {api_key}) — avoid putting keys in shared URLs.

      GET
      /api/news/feed
    

    
SQLite news_feed_items. Without a key: public list. With {api_key}: rows ranked like the Alerts page using your saved homes; add house_name to scope to one nickname.

    

      
        
Query / headerMeaning

      
      
        
api_key (optional)Query alternative; header preferred.

        
X-API-Key or Authorization: Bearer …Personal key from Generate.

        
house_nameSaved profile nickname (required match when using a key for personalization).

        
limit, offset, sourcePagination and source filter (same as public).

      
    

    
Examples

    

      
/api/news/feed?limit=15&offset=0

    

    

      
GET /api/news/feed?limit=20&house_name={house_name}
Header: X-API-Key: {api_key}
