Technical reference

True Threat Detection Score (TDS)

This page is for the exact math behind the 0–100 risk number. The rest of Hurricane Hub uses plain language on purpose. Nothing here replaces NWS warnings or county evacuation orders.

Formal aggregation

TDS = min (100, (Σi wi · si²) · Z)

  • si ∈ [0, 10] — sub-score for hazard i, from a sigmoid (logistic) transform of physical inputs.
  • wi — non-negative weights (Tampa Bay defaults sum to 1 before squaring).
  • Squaring si ensures one very large hazard term dominates the sum.
  • Z — evacuation-zone multiplier from official GIS polygons (letters A–E where available).

Sigmoid sub-score

s = 10 / (1 + exp(−k(x − m)))

x = measured input (e.g. gust mph, water anomaly); m = inflection; k = steepness.

API: home risk in one call

Signed-in users can pull the same bundle the Home risk page uses — no chaining dashboard + regional endpoints by hand.

Full bundle (includes dashboard, tampa_bay_regional, risk_card):

GET /api/assessment/home?address=500+Channelside+Dr,+Tampa+FL
POST /api/assessment/home  Content-Type: application/json
{"address": "500 Channelside Dr, Tampa FL"}

Coordinates (no geocoder):

GET /api/assessment/home?lat=27.9506&lon=-82.4572

Compact (scores + summaries only, no full metric trees):

GET /api/assessment/home?address=Tampa+FL&compact=1
POST /api/assessment/home
{"lat": 27.95, "lon": -82.45, "compact": true}

Responses include schema (…full.v1 or …compact.v1) for clients that version payloads.
